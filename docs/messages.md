# Отправка сообщений

Методы `send()` и `cost()` используют общий механизм подготовки данных и одни и
те же правила нормализации номеров.

## `send()`

Метод поддерживает два режима вызова.

### Один текст на один или несколько номеров

```python
from smsru_api import Client

smsru = Client("YOUR_API_KEY")
response = smsru.send(
    "79990000000",
    "79990000001",
    message="Код подтверждения: 1234",
)
```

### Разные тексты на разные номера

```python
from smsru_api import Client

smsru = Client("YOUR_API_KEY")
response = smsru.send(
    multi={
        "79990000000": "Ваш код: 1234",
        "79990000001": "Ваш код: 5678",
    }
)
```

Если передан `multi`, параметры `numbers` и `message` игнорируются.

Ключи `multi` нормализуются так же, как и обычные номера. Например, записи
`+7 (999) 000-00-00` и `8 (999) 000-00-00` считаются конфликтующими и
приведут к `ValueError`, потому что после нормализации это один и тот же
номер.

## Параметры `send()`

- `numbers` — один или несколько номеров телефона
- `message` — общий текст сообщения
- `multi` — словарь `{номер: текст}` для индивидуальных сообщений
- `from_name` — имя отправителя
- `ip_address` — IP-адрес конечного пользователя
- `timestamp` — Unix timestamp для отложенной отправки
- `ttl` — срок жизни сообщения в минутах, от `1` до `1440`
- `day_time` — учитывать локальное дневное время получателя
- `test` — включить тестовый режим API
- `debug` — включить локальный debug-режим; если `test` не задан, будет
  отправлен `test=1`
- `translit` — включить транслитерацию
- `partner_id` — переопределить партнерский идентификатор

## Валидация и ограничения

- За один запрос допускается не более 100 получателей.
- Если передать больше 100 номеров или элементов в `multi`, будет выброшено
  `OutOfPhoneNumbers`.
- Если `message` пустой и `multi` не передан, будет выброшен `ValueError`.
- Если список `numbers` пустой и `multi` не передан, будет выброшен
  `ValueError`.
- Если `ttl` меньше `1` или больше `1440`, будет выброшен `OutOfTimestamp`.
- Если `timestamp` выходит за допустимое окно отложенной отправки, будет
  выброшен `OutOfTimestamp`.
- Если `ip_address` не проходит проверку `ipaddress.ip_address`, будет
  выброшен `ValueError`.
- Если номер после нормализации пустой или не соответствует ожидаемому
  формату, будет выброшен `ValueError`.
- Если два ключа `multi` после нормализации дают один и тот же номер, будет
  выброшен `ValueError`.

## Нормализация номеров

При работе через `numbers` библиотека:

- удаляет все нецифровые символы
- удаляет ведущий код `+7` или `8`
- проверяет, что результат не пустой и содержит корректный 10-значный номер

Это позволяет передавать номера в разных распространенных форматах. Для
`multi` ключи словаря проходят ту же нормализацию и базовую валидацию, что и
обычные номера. Если после нормализации два ключа `multi` схлопываются в один
номер, библиотека считает это ошибкой входных данных.

## `cost()`

Метод оценивает стоимость сообщения без фактической отправки:

```python
from smsru_api import Client

smsru = Client("YOUR_API_KEY")
response = smsru.cost("79990000000", message="Проверка стоимости")
```

`cost()` использует ту же нормализацию номеров и те же базовые проверки, что и
`send()`, но не принимает параметры `multi`, `ttl`, `debug` и другие опции
отправки.

## `status()`

Получение статуса отправленного сообщения:

```python
from smsru_api import Client

smsru = Client("YOUR_API_KEY")
response = smsru.status("1000-100000")
```

Пример типичного ответа:

```json
{
  "status": "OK",
  "status_code": 100,
  "sms": {
    "1000-100000": {
      "status": "OK",
      "status_code": 102,
      "status_text": "Сообщение отправлено (в пути)"
    }
  }
}
```
